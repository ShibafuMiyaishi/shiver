# タスク完了時のチェックリスト（v3.2）

## コード変更後
1. エラーがないことを確認（ブラウザコンソール / サーバーログ）
2. テストが必要な場合は `tmp/` にテストコードを作成して実行
3. `docs/TASKS.md` のタスク状態を更新
4. 変更ファイルをgit add → 日本語コミットメッセージ → push

## Phase完了時
1. 技術書 Section 18 の完了条件を全て検証
2. `docs/TASKS.md` のPhase状態を更新
3. `CLAUDE.md` の「現在: Phase N」を更新
4. メモリ・ドキュメントを最新化（古い記述は削除）

## Phase 2 パーツ生成品質チェック（v3.2追加）
- [ ] 17パーツが全て正常に生成されるか
- [ ] キャラの外見（髪色・目の色）が各パーツで維持されているか
- [ ] 隠れ部分補完（前髪下の額など）が自然か
- [ ] 全パーツ重ね合わせ時に色ズレ・スタイル崩れがないか
- 上記4つが満たされれば Gemini 2.5 Flash継続。不満なら3 Proへアップグレード

## 絶対ルール（毎回確認）
- 設計を独断で変更していないか（技術書v3.2が最優先）
- 環境変数がハードコードされていないか
- `.env` がステージングに含まれていないか
- 非同期処理にtry-catchがあるか
- パーツ生成プロンプトにグリーンバック指定があるか（白背景禁止）
- Gemini APIは.aio(async client)を使っているか
